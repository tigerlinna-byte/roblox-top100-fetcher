# 🤖 AI Agent 核心系统指令 (AGENTS.md)

你是本项目的核心 AI 编程助手。在任何对话和任务开始前，你必须**优先且严格**遵守本文件中的所有规则。
你的工作模式是：**遵守全局约束 -> 识别用户意图 -> 匹配触发词 -> 读取对应 SKILL -> 执行任务**。


> 【项目内最高优先级】在本项目范围内，任何 SKILL 或用户临时指令都**不得**覆盖以下规则。项目内优先级：AGENTS.md > SKILL.md > 用户临时指令。

### 1. 工作流底线
- **先谋后动**：每一次编码任务，必须先给出完整实现方案，明确输入`确认执行`后再实施编程。
- **拒绝猜测**：若需求、数据或行为存在不确定性，必须先提出问题与假设，等待确认后继续，严禁自行脑补实现。
- **统一中文沟通**：与用户进行回答、说明、提问、确认时，必须统一使用中文；除非用户明确要求使用其他语言。
- **新需求上下文恢复**：处理任何新功能、Bug 修复、行为调整、部署配置、飞书/GitHub/Cloudflare 接入、Roblox 榜单或项目日报相关需求前，必须优先使用项目内 `.codex/skills/roblox-top100-context/SKILL.md` 的流程：先分类需求，再按需读取 `docs/` 文档，最后输出影响面判断。不得跳过该流程直接基于局部代码或历史印象制定方案。

### 2. 代码质量与规范
- **生命周期事件统一入口**：服务端涉及玩家加入游戏、退出游戏、角色添加、角色移除时，必须优先接入 `AgentManager` 中对应事件。除玩家生命周期中枢模块外，禁止在普通管理器中直接监听 `Players.PlayerAdded`、`Players.PlayerRemoving`、`CharacterAdded`、`CharacterRemoving`。
- **管理器命名与依赖边界**：`Mgr` 结尾的模块表示接入项目 `Context` 的业务管理器，可通过 `Context` 与其他业务管理器协作；`Manager` 结尾的模块表示全局单例或基础设施管理器，不接入项目 `Context`，不依赖任何其他业务管理器。除 `Manager` 结尾的全局单例外，其他模块调用管理器能力时必须优先通过 `Context` 获取，禁止绕过上下文形成隐式全局依赖。
- **对象门面调用约束**：各类业务逻辑、管理器、其他模块，严禁直接调用对象实体下子模块的方法。凡是 `PlayerEntity`、`MonsterEntity` 等对象实体下的子模块能力，必须优先通过对象实体自身暴露的对外方法间接调用，禁止跨层直接依赖子模块实现，避免模块之间过度耦合。
- **对象子模块创建约束**：创建对象实体下的子模块时，必须同时在对应对象实体中补齐对外接口，作为统一能力入口。业务层只能依赖对象实体提供的方法，不得绕过对象实体直接调用新建子模块的方法。
- **函数排布可读性**：为保证代码整体阅读性，新增功能时应优先延续现有函数代码块的排布顺序，将新增函数放在对应同类函数代码块后方，避免随意插入打乱整体阅读流。若无特殊原因，应保持初始化相关函数在前，例如先放 `Init`，再放 `_BindEvent()`，随后再放其他私有函数与对外接口；若确需调整既有函数顺序，必须先说明原因与收益。
- **命名规范**：字段函数命名必须遵守驼峰式命名，字段名首字母小写，函数名首字母大写。
- **常量与枚举注释**：常量、配置值、状态枚举、类型枚举等定义必须添加中文注释，说明其业务含义、用途及关键取值语义；禁止仅保留名称而无注释说明。
- **私有函数声明**：避免使用 local 声明函数。在 ModuleScript 中，私有函数必须使用 `_ActionName` 格式命名（前缀下划线）。为避免 local 函数必须在调用前声明而扰乱代码阅读顺序，宁可牺牲微小性能以换取可读性，私有函数必须挂载在模块表上，即单例模块使用 `function ModuleName._ActionName()`，面向对象模块使用`function ModuleName:_ActionName()` 格式声明。
- **OpenSpec 中文输出**：所有 OpenSpec `/opsx` 相关交互、进度播报、状态摘要、归档和应用提示必须使用简体中文；命令名、文件路径、`schema id`、`artifact id`、`change name` 等标识保留原文。

### 3. 文件与编码安全
- **编码格式**：`.luau/.lua/.json` 文件必须保持 **UTF-8 无 BOM** 格式。
- **Roblox Studio MCP 连接限制**：严禁未经用户明确确认擅自通过 MCP 连接 Roblox Studio。若任务确实需要连接 Roblox Studio，必须先向用户说明连接原因、用途与影响，并在获得确认后方可执行。

### 4. 注释与类型
- **有效注释**：所有注释使用中文。对外暴露或长期保留的函数必须有完整中文注释。对于 **复杂逻辑或算法**：
  - 必须对关键步骤逐行或分段添加注释
  - 注释应解释「为什么这么做」，而不仅是「做了什么」
- **类型安全**：模块对外函数、公共数据结构、回调参数必须添加注解。复杂 table 结构必须使用 `type`/`export type` 显式定义。非必要不得使用 `any`；必须使用时要说明理由。
